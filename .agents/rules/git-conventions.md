# Git Conventions

- **Atomic Commits:** You must commit changes in small, logical, atomic units.
- **Commit Messages:** You must write clear, descriptive commit messages starting with an action verb (e.g., "Add Zod validation to listing creation API").
- **No Secrets:** You must never commit `.env` files, passwords, or API keys. Reference `.env.example` instead.
